Module 1 — Branch Location APIs

Register employees, store office GPS coordinates, and track live location

GET /branch-location
Fetch all branch location records. Supports multiple query filters.
Query Parameters
ParameterTypeRequiredDescription
branchLocationIdnumberOptionalFetch specific record by ID
staffIdnumberOptionalFilter by staff/employee ID
branchIdnumberOptionalFilter by branch ID
trackingStatusstringOptionalactive | reached | absent | inactive
datestringOptionalFilter by date — format: YYYY-MM-DD
isActivebooleanOptionalFilter active/inactive records
Example Request
// Fetch Ravi's branch location on 15 May 2026 GET /branch-location?staffId=5&date=2026-05-15
Example Response
{ "data": [{ "branchLocationId": 1, "staffId": 5, "employeeName": "Ravi Kumar", "branchName": "K3 Office", "registeredLatitude": 10.790245, "registeredLongitude": 78.704512, "liveLatitude": 10.790260, "liveLongitude": 78.704500, "trackingStatus": "reached", "arrivalTime": "2026-05-15 09:15:00", "locationRadius": 100 }], "error": false, "message": "Success", "code": 200 }
POST /branch-location
Register an employee with their assigned office branch and GPS coordinates.
Request Body
FieldTypeRequiredDescription
staffIdnumberRequiredStaff ID from staffs table
employeeNamestringOptionalAuto-filled from staffs table if not given
branchIdnumberOptionalBranch ID from branches table
branchNamestringOptionalAuto-filled from branches table if not given
registeredLatitudenumberOptionalOffice GPS latitude — e.g. 10.790245
registeredLongitudenumberOptionalOffice GPS longitude — e.g. 78.704512
rolestringOptionalEmployee designation / role
officeEntryTimestringOptionalScheduled office entry time — HH:MM
locationRadiusnumberOptionalGeofence radius in metres (default: 100)
fcmTokenstringOptionalEmployee Firebase device token
Example Request Body
{ "staffId": 5, "branchId": 3, "registeredLatitude": 10.790245, "registeredLongitude": 78.704512, "role": "Sales Executive", "officeEntryTime": "09:00", "locationRadius": 100, "fcmToken": "eXQp3Fg...device_token" }
PATCH /branch-location/:branchLocationId/live-location
★ Core Tracking API — Receives live GPS from mobile app. Automatically calculates distance, detects office arrival, saves entry log, and fires push notification to employee + admin.
🔔 Automatic Push Notification: When distance ≤ locationRadius, this endpoint automatically saves an entry log and sends FCM notifications to both the employee and admin simultaneously.
Request Body
FieldTypeRequiredDescription
liveLatitudenumberRequiredEmployee current GPS latitude
liveLongitudenumberRequiredEmployee current GPS longitude
fcmTokenstringOptionalEmployee Firebase device token
adminFcmTokenstringOptionalAdmin Firebase device token (receives notification too)
mobileModelstringOptionalDevice model — e.g. "Samsung Galaxy A54"
batteryLevelstringOptionalBattery percentage — e.g. "78" or "78%"
Example Request
PATCH /branch-location/1/live-location { "liveLatitude": 10.790260, "liveLongitude": 78.704500, "mobileModel": "Samsung Galaxy A54", "batteryLevel": "78", "fcmToken": "<employee_device_token>", "adminFcmToken": "<admin_device_token>" }
Push Notification Sent (FCM)
{ "title": "Employee Reached Office", "body": "Ravi Kumar has arrived at K3 Office · 09:15 AM", "data": { "employeeName": "Ravi Kumar", "officeName": "K3 Office", "entryTime": "2026-05-15 09:15:00", "distanceMetres": "18.00", "type": "location_reached" } }
PUT /branch-location/:branchLocationId
Update any field of a branch location record — name, GPS, role, entry time, status, etc.
Example Request Body
{ "role": "Senior Sales Executive", "officeEntryTime": "08:30", "locationRadius": 150, "isActive": true }
DELETE /branch-location/:branchLocationId
Permanently delete a branch location tracking record.
Example Response
{ "data": { "branchLocationId": 1, "deleted": true }, "error": false, "message": "Success" }

Geofence Detection & Notification Flow

Step-by-step backend logic when PATCH live-location is called

📱 Scenario: Ravi reaches K3 Office

1
Mobile App sends live GPS

PATCH /branch-location/1/live-location → { liveLatitude: 10.790260, liveLongitude: 78.704500, mobileModel: "Samsung A54", batteryLevel: "78%" }

↓
2
Backend loads registered office coordinates

K3 Office: Latitude 10.790245 · Longitude 78.704512 · Radius 100 metres

↓
3
Haversine Distance Calculation

distance = haversine(10.790245, 78.704512, 10.790260, 78.704500) = 18 metres

↓
4
Geofence Condition Check

18m ≤ 100m → TRUE → Employee has reached the office ✓

↓
5
Update branch_locations record

tracking_status = "reached" · arrival_time = "2026-05-15 09:15:00"

↓
6
Save Entry Log to branch_location_entry_logs

employee_name: "Ravi Kumar" · office_name: "K3 Office" · entry_time: "09:15 AM" · date: "2026-05-15" · distance_metres: 18.00 · mobile_model: "Samsung A54" · battery_level: "78%" · status: "Location Reached"

↓
7
Firebase FCM Multicast — Employee + Admin

title: "Employee Reached Office" · body: "Ravi Kumar has arrived at K3 Office · 09:15 AM"
Sent to: [employee_device_token, admin_device_token] simultaneously

↓
8
Update notification_status on entry log

notification_status = "sent" (or "failed" / "pending") · notification_sent_at = timestamp

Module 2 — Branch Location Entry Log APIs

Auto-created arrival records — view and manage office entry events

GET /branch-location-entry-log
Fetch employee arrival / entry log records. Automatically populated when an employee reaches their registered office.
Query Parameters
ParameterTypeRequiredDescription
entryLogIdnumberOptionalFetch specific entry log by ID
staffIdnumberOptionalFilter by employee ID
branchIdnumberOptionalFilter by branch ID
branchLocationIdnumberOptionalFilter by parent branch location ID
employeeNamestringOptionalPartial name search (LIKE)
datestringOptionalExact date filter — YYYY-MM-DD
fromDatestringOptionalDate range start — YYYY-MM-DD
toDatestringOptionalDate range end — YYYY-MM-DD
statusstringOptionale.g. "Location Reached"
notificationStatusstringOptionalsent | failed | pending | not_sent
Response Fields
FieldDescription
entryLogIdEntry log primary key
employeeNameEmployee full name
officeNameOffice / branch name where arrived
entryTimeExact arrival timestamp
entryDateArrival date (YYYY-MM-DD)
distanceMetresDistance in metres when geofence triggered
mobileModelEmployee device model
batteryLevelBattery % at time of arrival
statusLocation Reached
notificationStatussent | failed | pending | not_sent
notificationSentAtTimestamp when FCM was fired
Example Request
GET /branch-location-entry-log?employeeName=Ravi&date=2026-05-15 GET /branch-location-entry-log?staffId=5&fromDate=2026-05-01&toDate=2026-05-31 GET /branch-location-entry-log?notificationStatus=failed
Example Response
{ "data": [{ "entryLogId": 1, "employeeName": "Ravi Kumar", "officeName": "K3 Office", "entryTime": "2026-05-15 09:15:00", "entryDate": "2026-05-15", "distanceMetres": 18.00, "mobileModel": "Samsung Galaxy A54", "batteryLevel": "78%", "status": "Location Reached", "notificationStatus": "sent", "notificationSentAt": "2026-05-15 09:15:01" }], "error": false, "message": "Success" }
DELETE /branch-location-entry-log/:entryLogId
Remove a specific entry log record by ID.
Example Response
{ "data": { "entryLogId": 1, "deleted": true }, "error": false, "message": "Success" }

Module 3 — Employee Tracking Report APIs

Session-level KM reports — device details, total distance, daily tracking sessions

GET /employee-tracking-report
Fetch employee tracking session reports. Filter by employee name, date, or date range.
Query Parameters
ParameterTypeRequiredDescription
reportIdnumberOptionalFetch specific report by ID
staffIdnumberOptionalFilter by staff ID
employeeNamestringOptionalPartial name search (LIKE match)
datestringOptionalFilter by exact date — YYYY-MM-DD
fromDatestringOptionalDate range start
toDatestringOptionalDate range end
Example Request
GET /employee-tracking-report?employeeName=Ravi&date=2026-05-15
GET /employee-tracking-report/km-summary
Aggregates total KM traveled per employee. Groups sessions and returns total distance, session count, and duration.
Query Parameters
ParameterDescription
staffIdFilter by staff ID
employeeNamePartial name match
dateSingle day summary
fromDate / toDateDate range summary
Example Response
{ "data": [{ "staffId": 5, "employeeName": "Ravi Kumar", "totalSessions": 22, "totalKmTraveled": 418.500, "firstTrackingDate": "2026-05-01", "lastTrackingDate": "2026-05-15", "totalDurationMinutes": 5280, "branchName": "K3 Office" }] }
POST /employee-tracking-report
Save a complete daily tracking session — KM traveled, device info, start and end times.
Request Body
FieldTypeRequiredDescription
staffIdnumberOptional*Staff ID (employeeName auto-filled from table)
employeeNamestringOptional** staffId or employeeName must be given
trackingDatestringOptionalYYYY-MM-DD — defaults to today
totalDistanceKmnumberOptionalTotal KM traveled in the session
mobileModelstringOptionalDevice model — e.g. "Samsung Galaxy A54"
batteryLevelstringOptional"78" or "78%" — normalised automatically
trackingStartTimestringOptionalISO datetime — e.g. "2026-05-15T09:00:00"
trackingEndTimestringOptionalISO datetime — e.g. "2026-05-15T17:30:00"
Example Request Body
{ "staffId": 5, "trackingDate": "2026-05-15", "totalDistanceKm": 42.750, "mobileModel": "Samsung Galaxy A54", "batteryLevel": "78", "trackingStartTime": "2026-05-15T09:00:00", "trackingEndTime": "2026-05-15T17:30:00" }
PUT /employee-tracking-report/:reportId
Update any field of an existing tracking session record.
Example Request Body
{ "totalDistanceKm": 45.200, "trackingEndTime": "2026-05-15T18:00:00", "batteryLevel": "32" }
DELETE /employee-tracking-report/:reportId
Permanently remove a tracking session report by ID.
Example Response
{ "data": { "reportId": 10, "deleted": true }, "error": false, "message": "Success" }

Firebase FCM Setup

Required to activate push notifications

Configuration Steps

1
Create Firebase Project

Go to console.firebase.google.com → Create project → Enable Cloud Messaging

↓
2
Get Server Key

Firebase Console → Project Settings → Cloud Messaging → Server Key (Legacy)

↓
3
Set Environment Variable on Server

Windows: set FCM_SERVER_KEY=AAAA...your_server_key_here
Linux/Mac: export FCM_SERVER_KEY=AAAA...your_server_key_here

↓
4
Pass device tokens in PATCH request

fcmToken = employee device token · adminFcmToken = admin device token
Both receive the same notification simultaneously via FCM multicast.
