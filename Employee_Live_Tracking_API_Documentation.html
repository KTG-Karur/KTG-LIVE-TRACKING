Employee Reached Office

Mobile App  ·  POST /office-arrival  ·  PATCH /branch-location/:id/live-location
Two approaches are available. Use POST /office-arrival for a single-step direct submission. Use PATCH /live-location for continuous real-time GPS pings (recommended for background tracking apps).
POST /office-arrival 📱 Mobile App

Employee submits current GPS coordinates. Server checks Haversine distance against branch coordinates. Logs arrival and sends FCM push notification to all admins of that branch.

Request Body
FieldTypeRequiredDescription
staffIdnumberRequiredEmployee staff ID
branchIdnumberRequiredBranch to check against
liveLatitudenumberRequiredCurrent GPS latitude
liveLongitudenumberRequiredCurrent GPS longitude
mobileModelstringOptionalDevice model e.g. "Samsung A54"
batteryLevelstringOptionalBattery % e.g. "78" or "78%"
fcmTokenstringOptionalEmployee FCM device token
entryTimestringOptionalISO datetime (defaults to server time)
Business Logic Flow
StepAction
1Validate staff ID exists in staffs table
2Fetch branch coordinates (latitude, longitude, allowed_radius)
3Calculate Haversine distance between employee GPS and branch GPS
4Set status: "Location Reached" (within radius) or "Outside Radius"
5Save record to branch_location_entry_logs table
6If within radius → fetch all admin FCM tokens → send multicast FCM push
7Update notification_status on entry log (sent / failed)
Sample Response
{
  "data": {
    "entryLogId": 42,
    "employeeName": "Ravi Kumar",
    "branchName": "K3 Office",
    "entryTime": "2026-05-15 09:15:00",
    "distanceMetres": 47.32,
    "allowedRadius": 100,
    "withinRadius": true,
    "status": "Location Reached",
    "batteryLevel": "78%",
    "notificationStatus": "sent",
    "notificationResult": { "success": true, "successCount": 2 }
  },
  "error": false, "message": "Success", "code": 200
}
PATCH /branch-location/:branchLocationId/live-location 📱 Mobile App (Real-time)

Continuous GPS ping. Updates live coordinates and auto-triggers geofence check. Fires FCM notification exactly once when employee enters the radius (tracking_status transitions to "reached").

Request Body
FieldTypeRequiredDescription
liveLatitudenumberRequiredCurrent GPS latitude
liveLongitudenumberRequiredCurrent GPS longitude
fcmTokenstringOptionalEmployee FCM token (refresh if changed)
adminFcmTokenstringOptionalAdmin FCM token to notify
mobileModelstringOptionalDevice model name
batteryLevelstringOptionalBattery percentage
📌 Push notification example: "Ravi Kumar has arrived at K3 Office · 09:15 AM"

FCM Push Notification — Admin Alert
When an employee arrives within the branch radius, all admin-role users assigned to that branch automatically receive a FCM push notification:

Title: Employee Reached Office
Body: "[EmployeeName] has reached [BranchName] at [HH:MM AM/PM]"

The notification payload also carries: entryLogId, staffId, officeName, entryDate, distanceMetres, mobileModel, batteryLevel.

Dashboard – Comparing Changes
Admin Panel  ·  Base URL: /dashboard

Returns a full dashboard summary including period-over-period comparisons for Employee Reached, KM Travelled, and Loan counts. Every comparable field follows a consistent { current, previous, change, trend } shape so the frontend can render trend arrows and percentage badges without any extra calculation.

GET /dashboard 🔒 JWT Required
Query Parameters
ParameterTypeRequiredDescription
branchIdnumber / comma-listOptionalFilter by one or more branch IDs e.g. 1 or 1,2,3
Compare Shape (used for all comparable fields)
{
  "current": <number>,   // value for current period
  "previous": <number>,   // value for previous period
  "change": <number>,   // % change e.g. 66.67 or -33.33
  "trend": "up" | "down" | "same"
}
